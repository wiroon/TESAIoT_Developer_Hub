# ควบคุม LED ผ่าน IPC (LED Control via IPC)

ตัวอย่างนี้แสดงวิธีที่คอร์แสดงผล CM55 สามารถควบคุมฮาร์ดแวร์บนคอร์ CM33 ผ่านคำสั่ง IPC ปุ่มบนหน้าจอสัมผัส CM55 ส่ง `IPC_CMD_GPIO_LED` (0x80) ไปยัง CM33 ซึ่งสลับ GPIO LED บนบอร์ด

CM55 ไม่สามารถเข้าถึงขา GPIO ที่กำหนดให้ CM33 ได้โดยตรง เนื่องจาก protection policy controller (PPC) ของ PSoC Edge แทนที่จะเข้าถึงตรง มันส่งคำสั่ง IPC ขอให้ CM33 ทำการดำเนินการ GPIO นี่คือรูปแบบการเข้าถึงอุปกรณ์ต่อพ่วงข้ามคอร์มาตรฐาน

LVGL LED widget บนหน้าจอสะท้อนสถานะ LED จริง สลับระหว่างเขียว (เปิด) และเทาเข้ม (ปิด) ทุกครั้งที่กดปุ่ม สถานะถูกติดตามแบบ local บน CM55 และส่งไปยัง CM33 ผ่าน data field ของ IPC message

IPC message พกไบต์เดียว: 0x01 สำหรับ LED เปิด และ 0x00 สำหรับ LED ปิด payload ขั้นต่ำนี้สาธิตรูปแบบ command ที่ CM55 ทำหน้าที่เป็นตัวควบคุม (ตัดสินใจ UI) และ CM33 ทำหน้าที่เป็นตัวดำเนินการ (เข้าถึงฮาร์ดแวร์)
