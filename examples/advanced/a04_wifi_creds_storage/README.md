# จัดเก็บข้อมูล WiFi แบบถาวร (WiFi Credentials Storage)

ตัวอย่างนี้สาธิตการจัดการข้อมูลรับรอง WiFi แบบถาวรโดยใช้ LittleFS บน QSPI Flash ผู้ใช้สามารถบันทึกเครือข่าย WiFi ได้สูงสุด 5 รายการพร้อม SSID และรหัสผ่าน ดูรายการเครือข่ายที่บันทึกแบบเลื่อนได้ ลบรายการทีละรายการ และสลับพฤติกรรมเชื่อมต่ออัตโนมัติ ข้อมูลที่บันทึกแต่ละรายการมี CRC32 Checksum เพื่อตรวจสอบความถูกต้องของข้อมูล

ชั้นการจัดเก็บใช้ qspi_wifi_creds API ซึ่งเขียนลง QSPI Flash ผ่านระบบไฟล์ LittleFS ทำให้ข้อมูลรับรองคงอยู่แม้ปิดเครื่องหรืออัปเดตเฟิร์มแวร์ (ตราบที่ไม่ได้ลบ QSPI Partition) CRC32 จะถูกคำนวณใหม่ทุกครั้งที่บันทึกและตรวจสอบทุกครั้งที่โหลดเพื่อตรวจจับความเสียหายของ Flash

UI ประกอบด้วย 3 ส่วน: วิดเจ็ต List แสดงเครือข่ายที่บันทึกพร้อมปุ่มลบ, ช่องป้อนข้อมูลสำหรับเพิ่มข้อมูลรับรองใหม่ (SSID Textarea และ Password Textarea) และแถวควบคุมพร้อมปุ่มเพิ่มและปุ่มสลับเชื่อมต่ออัตโนมัติ แต่ละรายการใน List แสดง SSID และปุ่มไอคอนถังขยะสำหรับลบ

เมื่อเปิดใช้การเชื่อมต่ออัตโนมัติ เฟิร์มแวร์จะพยายามเชื่อมต่อเครือข่ายที่บันทึกไว้ตามลำดับเมื่อบูต โดยหยุดที่การเชื่อมต่อสำเร็จครั้งแรก ซึ่งเป็นพฤติกรรมเดียวกับ WiFi Manager ของเฟิร์มแวร์จริง และเป็นตัวอย่างอ้างอิงสำหรับรูปแบบการจัดเก็บข้อมูลรับรองแบบถาวร
