# ระบบอินพุตเกมแบบรวม (Unified Game Input)

ตัวอย่างนี้สร้างชั้นการจัดการอินพุตแบบรวมที่ผสานอินพุตจากจอยสติ๊ก USB HID F310 และ D-pad แบบสัมผัสบนหน้าจอเข้าเป็นโครงสร้าง game_input_state_t เดียว ทำให้ได้ API อินพุตที่สม่ำเสมอสำหรับตัวอย่างเกม ไม่ว่าผู้ใช้จะมีคอนโทรลเลอร์จริงหรือใช้หน้าจอสัมผัส

จอยสติ๊ก F310 สื่อสารผ่าน IPC Bridge: CM55 Core อ่านรายงาน USB HID ผ่าน SEGGER emUSB-Host Stack และส่งสถานะจอยสติ๊กไปยัง CM33_NS Application Core โดยใช้ IPC_CMD_JOYSTICK_STATE (0xC0) ข้อความ IPC ประกอบด้วยค่าแกนอนาล็อกและสถานะปุ่มที่บรรจุในรูปแบบไบนารีขนาดกะทัดรัด

D-pad แบบสัมผัสวาดพื้นที่สัมผัสทิศทาง 4 ทิศและปุ่มแอคชัน 2 ปุ่มบนหน้าจอ LVGL เหตุการณ์สัมผัสถูกจับผ่าน LVGL Input Device Callback และแปลงเป็น Flag ทิศทางและปุ่มเดียวกับจอยสติ๊ก Overlay เป็นแบบกึ่งโปร่งใสเพื่อไม่ให้บดบังเนื้อหาเกม

โครงสร้าง game_input_state_t มี Flag แบบ Boolean สำหรับ up/down/left/right/a/b/start พร้อมค่าอนาล็อก X/Y (ช่วง 0-255 ค่ากลาง 128) แหล่งอินพุตทั้งสองถูกรวมแบบ OR ดังนั้นการกดขึ้นบนจอยสติ๊กหรือ D-pad แบบสัมผัสจะให้ผลลัพธ์เดียวกัน หน้าจอทดสอบแบบภาพแสดงสถานะอินพุตทั้งหมดแบบเรียลไทม์สำหรับการดีบัก
