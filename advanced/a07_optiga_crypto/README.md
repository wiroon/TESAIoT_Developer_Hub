# สาธิตการเข้ารหัสด้วย OPTIGA (OPTIGA Crypto Demo)

ตัวอย่างนี้สาธิตขั้นตอนการเข้ารหัสลับแบบครบวงจรโดยใช้ชิปรักษาความปลอดภัย OPTIGA Trust M ประกอบด้วย SHA-256 Hashing, ECDSA Signing และการตรวจสอบลายเซ็น ผู้ใช้ป้อนข้อความผ่าน LVGL Textarea และแอปพลิเคชันจะแสดงแต่ละขั้นตอนการเข้ารหัสพร้อมข้อมูลสถานะ

ขั้นตอนการทำงานมี 4 ระยะ ระยะแรก ข้อมูลจากผู้ใช้จะถูกแฮชด้วย SHA-256 ได้ Digest ขนาด 32 ไบต์แสดงในรูปแบบเลขฐานสิบหก ระยะที่สอง Digest ถูกเซ็นด้วย ECDSA โดยใช้กุญแจที่เก็บใน OPTIGA Key Slot 0xE0F1 (Slot 0xE0F0 ถูกบล็อกเพื่อความปลอดภัย) ระยะที่สาม ลายเซ็นที่เข้ารหัสแบบ DER จะถูกแสดง ระยะที่สี่ ลายเซ็นถูกตรวจสอบกับค่าแฮชโดยใช้กุญแจสาธารณะที่สอดคล้องกัน

แต่ละขั้นตอนจะอัปเดตแผงความคืบหน้าด้วยตัวบ่งชี้สีเขียว (ผ่าน) หรือสีแดง (ไม่ผ่าน) การดำเนินการเข้ารหัสทำงานใน FreeRTOS Task แยกต่างหากเพื่อไม่ให้กระทบการรีเฟรชหน้าจอ LVGL โปรโตคอลการใช้บัส I2C ร่วมกัน (หยุด/เริ่มหน้าจอสัมผัสผ่าน IPC) ถูกจัดการอย่างโปร่งใสภายในแต่ละการเรียก OPTIGA

ข้อควรพิจารณาด้านความปลอดภัยที่สำคัญ: กุญแจส่วนตัวไม่เคยออกจากชิป OPTIGA การเซ็นทั้งหมดเกิดขึ้นภายในชิปรักษาความปลอดภัย และเฉพาะไบต์ของลายเซ็นเท่านั้นที่ส่งกลับไปยัง MCU หลัก การเซ็นที่มีฮาร์ดแวร์รองรับให้การรับประกันความปลอดภัยที่แข็งแกร่งกว่าการเซ็นด้วยซอฟต์แวร์อย่างเดียว
