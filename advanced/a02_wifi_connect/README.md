# ขั้นตอนการเชื่อมต่อ WiFi (WiFi Connect Flow)

ตัวอย่างนี้สร้างระบบเชื่อมต่อ WiFi แบบ Station ที่สมบูรณ์พร้อมอินเทอร์เฟซผู้ใช้แบบหลายขั้นตอน การทำงานจะดำเนินผ่านหลายสถานะ: พร้อมใช้งาน, กำลังสแกนเครือข่าย, เลือก AP จาก Dropdown, ป้อนรหัสผ่านผ่านแป้นพิมพ์บนหน้าจอ, เชื่อมต่อพร้อม Spinner แสดงสถานะ และสุดท้ายแสดงหมายเลข IP ที่ได้รับ

ขั้นตอนการสแกนใช้ cy_wcm_start_scan พร้อม Callback ที่เก็บ Access Point ได้สูงสุด 20 รายการ เมื่อสแกนเสร็จ SSID ทั้งหมดจะถูกใส่ในวิดเจ็ต LVGL Dropdown ผู้ใช้เลือกเครือข่ายและพิมพ์รหัสผ่านลงใน Textarea ที่มี LVGL Keyboard แนบอยู่ ประเภทการเข้ารหัสจะถูกตรวจจับอัตโนมัติจากผลการสแกน

การเชื่อมต่อใช้ cy_wcm_connect_ap พร้อม Timeout ที่ปรับได้ (ค่าเริ่มต้น 15 วินาที) ระหว่างเชื่อมต่อ Spinner และ Label แสดงสถานะจะแจ้งความคืบหน้าให้ผู้ใช้ทราบ เมื่อสำเร็จจะแสดงหมายเลข IPv4 ที่ได้รับ เมื่อล้มเหลวจะแสดงข้อความผิดพลาดพร้อมรหัสผลลัพธ์ WCM และผู้ใช้สามารถลองใหม่ได้

รูปแบบ State Machine ที่ใช้ (IDLE, SCANNING, SELECTING, CONNECTING, CONNECTED, ERROR) เป็นแนวทางที่พร้อมใช้งานจริงสำหรับการจัดการ WiFi แบบ Asynchronous ควบคู่กับอินเทอร์เฟซ LVGL ที่ตอบสนองได้ดี การเปลี่ยนสถานะแต่ละครั้งจะอัปเดต UI และเปิดหรือปิดวิดเจ็ตที่เกี่ยวข้อง

รองรับทั้ง TESAIoT Dev Kit, PSoC Edge AI Kit และ PSoC Edge Eva Kit
