# ส่งข้อมูลเซ็นเซอร์ผ่าน MQTT (MQTT Sensor Publisher)

ตัวอย่างนี้เชื่อมต่อ WiFi, สร้าง Session กับ MQTT Broker และส่งข้อมูลเซ็นเซอร์ในรูปแบบ JSON จาก IMU BMI270 ทุก 5 วินาที หน้าจอ UI แสดงสถานะการเชื่อมต่อ จำนวนข้อความที่ส่ง และ JSON Payload ล่าสุดแบบเรียลไทม์

การเชื่อมต่อ MQTT ใช้ QoS 1 (รับประกันส่งอย่างน้อยหนึ่งครั้ง) พร้อม Clean Session หัวข้อ (Topic) ใช้รูปแบบ `tesaiot/{board_id}/sensors` ทำให้สามารถระบุข้อมูลจากบอร์ดหลายตัวบน Broker ได้อย่างง่ายดาย แต่ละข้อความประกอบด้วยค่าความเร่ง XYZ และ Timestamp แบบ Monotonic

สถาปัตยกรรมแยก WiFi Connection, MQTT Session Management และการอ่านเซ็นเซอร์ไว้ใน FreeRTOS Task เดียวที่ดำเนินผ่านสถานะที่กำหนดไว้ชัดเจน การอัปเดต UI ของ LVGL ใช้ lv_lock/lv_unlock เพื่อรับประกันความปลอดภัยของเธรดร่วมกับ Display Task

การจัดการข้อผิดพลาดครอบคลุม WiFi เชื่อมต่อล้มเหลว, MQTT Broker ไม่สามารถเข้าถึงได้, การส่งล้มเหลวพร้อมระบบลองใหม่ และเหตุการณ์ตัดการเชื่อมต่อผ่าน MQTT Event Callback ตัวจัดการการตัดเชื่อมต่อจะตั้งค่า Flag เพื่อกระตุ้นการเชื่อมต่อใหม่อัตโนมัติในรอบการส่งถัดไป

รองรับทั้ง TESAIoT Dev Kit, PSoC Edge AI Kit และ PSoC Edge Eva Kit เนื่องจากทั้งสองบอร์ดมี IMU BMI270 และความสามารถ WiFi
