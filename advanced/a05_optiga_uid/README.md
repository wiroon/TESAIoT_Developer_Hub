# อ่านหมายเลขเฉพาะจาก OPTIGA Trust M (OPTIGA Trust M UID Reader)

ตัวอย่างนี้สาธิตการอ่านหมายเลขเฉพาะ (UID) จากชิปรักษาความปลอดภัย Infineon OPTIGA Trust M โดย UID เป็นค่า 27 ไบต์ที่ถูกโปรแกรมจากโรงงานและเก็บอยู่ที่ Object ID 0xE0C2 ซึ่งระบุชิปแต่ละตัวได้อย่างไม่ซ้ำกัน เหมาะสำหรับการพิสูจน์ตัวตนอุปกรณ์และการติดตามทรัพย์สิน

ประเด็นสำคัญของตัวอย่างนี้คือโปรโตคอลการใช้บัส I2C ร่วมกัน OPTIGA Trust M ใช้ SCB0 (I2C) ร่วมกับตัวควบคุมหน้าจอสัมผัสของ CM55 ก่อนดำเนินการใดๆ กับ OPTIGA จะต้องหยุดไดรเวอร์หน้าจอสัมผัสของ CM55 ผ่านคำสั่ง IPC IPC_CMD_TOUCH_PAUSE (0xD6) หลังจากดำเนินการ OPTIGA เสร็จ IPC_CMD_TOUCH_RESUME (0xD7) จะกลับมาใช้หน้าจอสัมผัสได้ตามปกติ หากไม่ปฏิบัติตามโปรโตคอลนี้จะทำให้เกิดการแย่งบัส I2C และข้อมูลเสียหาย

ลำดับการเริ่มต้น OPTIGA จะเปิด Application Context บนชิปรักษาความปลอดภัย ซึ่งเกี่ยวข้องกับ Handshake ที่ความเร็ว I2C 100 kHz ค่า BSP Clock Divider ตั้งไว้ที่ 31 สำหรับความเร็วนี้ ความเร็วที่สูงกว่า (เช่น Divider=9 สำหรับ 400 kHz) จะทำให้ระบบแสดงผลและหน้าจอสัมผัสของ CM55 เสียหาย PAL (Platform Abstraction Layer) จะเริ่มต้น SCB0 Context ใหม่ขณะที่หน้าจอสัมผัสถูกหยุดไว้

UI แสดงความคืบหน้าเป็นขั้นตอน: กำลังเริ่มต้น OPTIGA, กำลังอ่าน UID และแสดงผลลัพธ์ UID ขนาด 27 ไบต์จะถูกจัดรูปแบบเป็นสตริงฐานสิบหกและแสดงใน Label ขนาดใหญ่แบบ Monospace ตัวบ่งชี้สถานะ (จุดสี) แสดงความคืบหน้าของแต่ละขั้นตอน

รองรับทั้ง TESAIoT Dev Kit, PSoC Edge AI Kit และ PSoC Edge Eva Kit เนื่องจากทั้งสองบอร์ดมี OPTIGA Trust M บน SCB0
